# IOTA6910K Assignment 1 DL（中文说明）

## 这是什么

这是一个面向 **distributed learning / distributed training system design** 的课程作业包。

这份作业**不是**让学生真的去训练一个大模型，也**不是**要求学生搭建真实的多机多卡集群。相反，它把问题简化成一个可以在个人笔记本上完成的“系统设计 + 轻量仿真”任务。

学生面对的是一个**虚拟的跨地域 GPU 集群环境**：

- 有多个区域（region）
- 每个区域里有不同类型的 GPU
- 不同区域之间带宽和延迟不同
- GPU 的速度和成本也不同

学生需要回答的问题很像真实系统设计中会遇到的问题：

- 应该用哪些集群？
- 每个集群用多少 GPU？
- 不同速度的 GPU 怎么分工？
- 训练时是所有节点直接同步，还是分层同步？
- 多久做一次全局同步更合适？

换句话说，这个作业的核心不是“把模型训练出来”，而是**设计一个合理的分布式训练方案，并解释为什么这样设计**。

## 为什么这样设计

老师给出的要求很明确：作业必须能够在学生自己的 laptop 上完成，不能依赖真实的 multi-node GPU infrastructure。

因此，这份 assignment 的设计思路是：

1. **保留真实分布式训练里的关键矛盾**
   - 计算快的 GPU 更贵
   - 跨区域通信慢
   - 同步更频繁，收敛可能更稳定，但通信成本更高
   - 用更多资源不一定更划算

2. **去掉不必要的工程门槛**
   - 不要求真实部署
   - 不要求真实大模型训练
   - 不要求多机环境
   - 不要求 GPU

3. **把重点放在系统 reasoning 上**
   - 资源选择是否合理
   - 同步结构是否合理
   - 伪代码是否清楚表达了系统行为
   - 仿真结果是否支撑设计结论

所以，这个作业本质上是在训练学生做一件很重要的事情：  
**把一个分布式学习问题抽象成系统设计问题，然后用模型化和仿真去分析 tradeoff。**

## 学生到底要做什么

学生主要完成四件事：

### 1. 设计一个 distributed training strategy

学生需要提出自己的训练方案，例如：

- 只用少数高性能区域，减少慢链路
- 用区域内先同步、再做跨区域同步的层次化方案
- 让快 GPU 承担更多工作，慢 GPU 承担更少工作
- 降低跨区域同步频率，换取更低通信代价

这里没有唯一正确答案，关键是方案要**讲得通**。

### 2. 写伪代码

学生不能只给一个口头想法，还要用伪代码把系统流程表达清楚，例如：

- worker 如何进行本地计算
- region 内如何聚合
- global aggregator 如何同步
- 不同层次之间多久通信一次

这部分占分很高，因为它反映学生是否真的理解自己提出的系统。

### 3. 在轻量 simulator 中实现自己的策略

学生不需要实现完整训练框架，只需要在给定入口中实现策略逻辑：

- 选择哪些 cluster
- 如何分配 GPU
- 使用哪种同步方式
- 设置多大的同步间隔

然后运行提供好的 baseline 和自己的 custom strategy，输出对比结果。

### 4. 分析结果

学生需要比较：

- 总训练时间
- 通信时间
- 通信量
- GPU-hour 成本
- 同步频率带来的 penalty

重点不是单纯找一个最小值，而是解释：

- 为什么你的方案更快 / 更便宜 / 更平衡
- 为什么某些 baseline 在某个场景下不合适
- 你的方案在哪些情况下会失效

## 这个作业为什么适合 distributed learning 课程

因为它考察的是 distributed learning 里非常核心的能力：

- **系统抽象能力**：把复杂训练环境抽象成可分析的问题
- **算法与系统协同思维**：训练策略和通信结构一起设计
- **性能建模意识**：不仅看 accuracy，也看时间、通信和成本
- **实验分析能力**：用仿真结果支撑论证，而不是只给直觉

它虽然不做真实训练，但研究的问题和真实大规模训练非常接近。

很多现实中的 distributed learning 系统设计，本质上也是在解决类似问题：

- 如何跨数据中心协作训练
- 如何处理异构硬件
- 如何平衡吞吐、同步开销和成本
- 如何在资源有限时找到合理的训练计划

## 目录怎么理解

### `student/`

这是发给学生的部分，包含：

- `assignment.md`：正式作业要求
- `scenarios/`：不同的虚拟集群场景
- `simulator/`：轻量仿真器
- `scripts/`：运行 baseline 和 custom strategy 的脚本
- `strategies/student_custom_strategy.py`：学生主要需要修改的文件

### `instructor/`

这是老师自己保留的部分，包含：

- 参考设计思路
- 参考策略实现
- 参考结果
- 评分 rubric
- 课程目标说明

## 难度大概怎么样

这份作业的难度是**中等偏上**，但难点主要不在编码量，而在于：

- 能不能提出合理的系统方案
- 能不能把伪代码写清楚
- 能不能从结果中读出 tradeoff
- 能不能说清楚设计的局限性

如果学生只会“跑脚本”，拿不到高分。  
如果学生能把方案、伪代码、结果和分析连成一条完整的逻辑链，就能做得很好。

## 一句话总结

这份 assignment 的目标，是让学生在**不依赖真实大规模硬件**的前提下，练习如何设计、表达、实现和分析一个 **geo-distributed distributed learning system strategy**。
